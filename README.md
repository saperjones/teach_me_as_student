# teach_me_as_a_student

一个 [Claude Code / Agent Skill](https://agentskills.io)，把复杂知识讲到 **一个中学生也能真正听懂、而不只是看懂字面**。

像好老师给零基础高中生上课那样：掰开揉碎、层层追问、大量生活化举例和类比、全程中文大白话，同时**不牺牲技术精确性**。

## 它能做什么

支持两类输入：

| 输入 | 处理方式 | 产出 |
|------|----------|------|
| **一篇文章**（Markdown / PDF / 文档） | 保留原文章节顺序，一章一章讲；每个复杂概念掰开揉碎，每个流程整理成宽表格 | 一篇 `Plain_` 开头的大白话版本（不改、不覆盖原文） |
| **一个概念**（如 Clothoid、Kalman Filter、Transformer、Diffusion Model） | 用同一套教学配方，覆盖"是什么 / 为什么提出 / 解决什么问题 / 没有它会怎样 / 工作原理 / 数学本质 / 完整例子 / 生活类比 / 常见误区 / 与相似概念区别 / 工程应用"等 12 点 | 一篇教材级讲解 |

核心讲解配方（两个场景共用）：

> 一句话定位 → 它要解决什么问题 → 打个生活化比方 → 手把手走一个例子 → 为什么这么设计（取舍）→ 常见误区/坑 → 一句话收尾

## 硬要求（通俗 ≠ 失真）

- 技术标识符（字段名、接口、参数、错误码等）一律用 `行内代码` 原样保留。
- 数值、阈值、范围原样保留。
- 带"必须 / 仅 / 禁止 / 默认"等限定词的约束力度一字不改。
- 不加料、不丢关键信息。

## 安装 / 使用

把 `teach_me_as_a_student/` 目录放进你的技能目录（例如 Claude Code 插件的 `skills/`），然后：

```
/teach_me_as_a_student <文章路径或概念名>
```

或直接说"用大白话把这篇/这个概念讲到中学生也能懂"。

## 目录结构

```
teach_me_as_a_student/
  SKILL.md      # 技能定义（frontmatter + 讲解方法 + 硬要求 + 写作风格）
req.txt         # 原始需求
```

## 由来

本技能由 `tech_doc_plain`（技术文档平实化、保精确、流程做成表格）与 `teach_me_as_student`（针对单个概念深入浅出讲解）两个技能合并而来，写作深度以一份真实的自动驾驶数据生产复盘文档为标杆。
