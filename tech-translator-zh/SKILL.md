# tech-translator-zh

Rewrite Chinese technical writing so it reads as if written by an experienced Chinese engineer — not translated from an English paper, not generated by AI. This Skill simplifies **language, not technology**.

## When to use

- Chinese text that reads like a word-for-word rendering of English academic terms (指代消解, 对齐, 幻觉, 嵌入, 检索…).
- Writing heavy with abstract nouns, nominalized verbs, management jargon, or AI filler.
- The user asks to 改写 / 润色 / 说人话 a Chinese tech doc, or starts input with `/tech-translator-zh`.

Do **not** use to simplify technical content, shorten documents, or translate between languages.

## Core Principles

Enforce in strict priority order. A lower principle never overrides a higher one.

0. Never change technical meaning.
1. Never remove information.
2. Never invent information.
3. Translate meaning, not terminology.
4. Prefer natural Chinese.
5. Minimize reading effort.

## Rewrite Algorithm

For each sentence:

1. **Detect** difficulty: literal English-paper translations, academic wording, abstract nouns, nominalized verbs, AI filler, management jargon.
2. **Recover intent** — ask: "what is the author actually trying to say?"
3. **Rewrite the whole sentence** in natural Chinese carrying the exact same meaning. Do not just gloss the term.
4. **Verify** technical meaning is unchanged (Principle 0).

Translate the sentence, not the vocabulary:

> 本模块首先完成 Grounding。 → 本模块会先根据真实资料确定它指的是谁。
> 系统采用 Retrieval-Augmented Generation。 → 系统回答前会先到知识库查资料，再结合这些资料生成答案。

## Rewrite Rules

- Prefer verbs over nouns; concrete actions over abstract concepts.
- Delete empty modifiers (有效地, 进一步, 相关的, 一定程度上, 众所周知…).
- Split long AI sentences into short ones — one idea each.
- Replace management jargon (赋能, 抓手, 闭环, 对齐一下) with the concrete action it stands for.
- Turn academic phrasing into everyday engineering speech.
- Keep an English term when engineers naturally say it in English (token, PR, commit, embedding 模型). Rewrite a term only when its Chinese literal translation is what causes the confusion.
- Explain a concept inline only when the reader cannot follow without it — and keep it brief.
- Never over-simplify: keep every technical distinction intact.
- Judge each term in context; do not rely on a fixed glossary.

## Anti-patterns

Never default to literal term-for-term translation:

- Grounding → ~~指代消解~~ → 根据真实资料确定它指的是谁
- Hallucination → ~~幻觉~~ → 一本正经地编出不存在的内容
- Embedding → ~~嵌入~~ → 把文字变成一串能比较相似度的数字
- Retrieval → ~~检索~~ → 到知识库里查资料

Rewrite the surrounding sentence naturally. Do not stack an explanation onto the jargon and leave the jargon leading.

## Output Rules

- Preserve all Markdown: headings, tables, code blocks, links, lists, inline formatting.
- Never alter code blocks, identifiers, or values — change prose only.
- Keep structure and technical accuracy; return the document in the same format as the input.

## Quality Checklist

Before returning, confirm every item:

- Technical meaning preserved; nothing removed or invented.
- Markdown intact.
- Every sentence clear in one read.
- Reads like natural Chinese from an experienced engineer.
- No academic literalism, no AI filler.
